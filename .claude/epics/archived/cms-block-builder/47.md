## Description

Register all new routes for the block editor and settings pages in `routes/web.php`, and add an "Editor" link to the admin sites index page so admins can navigate to the block editor for each site.

## Acceptance Criteria

- [ ] All 5 new routes are registered in the admin middleware group
- [ ] Routes are named correctly and follow existing naming conventions
- [ ] `php artisan route:list` shows all new routes without errors
- [ ] "Editor" link appears next to each site's Edit button on the sites index page
- [ ] All routes use proper HTTP methods (GET, PUT, POST)
- [ ] Routes are protected by the same auth/admin middleware as other admin routes

## Technical Details

### Route Definitions

File: `routes/web.php` (modify existing)

Add inside the existing admin middleware group:

```php
// Block Editor
Route::get('admin/sites/{site}/editor', [App\Http\Controllers\Admin\BlockEditorController::class, 'edit'])
    ->name('admin.sites.editor');
Route::put('admin/sites/{site}/blocks', [App\Http\Controllers\Admin\BlockEditorController::class, 'update'])
    ->name('admin.sites.blocks.update');
Route::post('admin/sites/{site}/blocks/upload', [App\Http\Controllers\Admin\BlockEditorController::class, 'uploadImage'])
    ->name('admin.sites.blocks.upload');

// Settings
Route::get('admin/settings', [App\Http\Controllers\Admin\SettingsController::class, 'edit'])
    ->name('admin.settings.edit');
Route::post('admin/settings', [App\Http\Controllers\Admin\SettingsController::class, 'update'])
    ->name('admin.settings.update');
```

### Route Summary

| Method | URI | Controller@Method | Route Name |
|--------|-----|-------------------|------------|
| GET | admin/sites/{site}/editor | BlockEditorController@edit | admin.sites.editor |
| PUT | admin/sites/{site}/blocks | BlockEditorController@update | admin.sites.blocks.update |
| POST | admin/sites/{site}/blocks/upload | BlockEditorController@uploadImage | admin.sites.blocks.upload |
| GET | admin/settings | SettingsController@edit | admin.settings.edit |
| POST | admin/settings | SettingsController@update | admin.settings.update |

### Sites Index Update

File: `resources/views/admin/sites/index.blade.php` (modify existing)

Add an "Editor" link next to each site's existing Edit button:

```blade
<!-- Next to the existing Edit button for each site -->
<a href="{{ route('admin.sites.editor', $site) }}"
   class="text-purple-600 hover:text-purple-900 mr-3"
   title="{{ __('Editor') }}">
    {{ __('Editor') }}
</a>
```

Or as a button/icon:

```blade
<a href="{{ route('admin.sites.editor', $site) }}"
   class="inline-flex items-center px-3 py-1 bg-purple-100 text-purple-700 rounded hover:bg-purple-200 text-sm">
    🧩 {{ __('Editor') }}
</a>
```

### Optional: Admin Sidebar Navigation

If the admin layout has a sidebar, add a "Settings" link:

```blade
<a href="{{ route('admin.settings.edit') }}" class="...">
    {{ __('Settings') }}
</a>
```

### Verification

After wiring routes, run:

```bash
php artisan route:list --name=admin.sites.editor
php artisan route:list --name=admin.sites.blocks
php artisan route:list --name=admin.settings
```

All 5 routes should appear with correct methods, URIs, and names.
