### Implementation Notes

- Read the existing JSON file content first, then merge the new keys
- Preserve alphabetical ordering if the existing file uses it
- Ensure valid JSON (no trailing commas, proper escaping)
- The `__()` function in Laravel looks up keys from these JSON files when the key matches the current locale

### Example Merge Pattern

```php
// Read existing
$existing = json_decode(file_get_contents('lang/en.json'), true);

// Add new keys (existing keys are NOT overwritten)
$newKeys = [
    'Block editor' => 'Block editor',
    'Add block' => 'Add block',
    // ...
];

$merged = array_merge($newKeys, $existing); // existing takes precedence
ksort($merged);

file_put_contents('lang/en.json', json_encode($merged, JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE));
```
