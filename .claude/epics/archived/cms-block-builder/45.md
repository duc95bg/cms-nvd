## Description

Implement the theme system: seed the database with predefined themes and default settings, update the site creation form to allow theme selection, and modify the site store logic to clone a theme's block preset into the new site's blocks when a theme is chosen.

## Acceptance Criteria

- [ ] `ThemeSeeder` seeds 3 themes with realistic blocks_preset data
- [ ] `SettingsSeeder` seeds default site settings (name, tagline, contact info)
- [ ] Both seeders are registered in `DatabaseSeeder`
- [ ] Seeders use `updateOrCreate` to be idempotent (safe to run multiple times)
- [ ] Site creation form shows theme selection as radio cards with thumbnail, name, and description
- [ ] "No theme" option is available (blank site, no blocks)
- [ ] When a theme is selected on site create, blocks_preset is cloned into `site.blocks`
- [ ] `site.theme_id` is set when a theme is chosen
- [ ] Site creation without a theme still works as before (no regression)

### SettingsSeeder

```php
namespace Database\Seeders;

use App\Models\Setting;
use Illuminate\Database\Seeder;

class SettingsSeeder extends Seeder
{
    public function run(): void
    {
        $defaults = [
            'site_name' => ['vi' => 'Tên website', 'en' => 'Site Name'],
            'tagline' => ['vi' => 'Slogan website', 'en' => 'Site Tagline'],
            'email' => 'admin@example.com',
            'phone' => '+84 123 456 789',
            'address' => ['vi' => 'Hà Nội, Việt Nam', 'en' => 'Hanoi, Vietnam'],
            'facebook' => '',
            'instagram' => '',
            'youtube' => '',
            'tiktok' => '',
        ];

        foreach ($defaults as $key => $value) {
            Setting::updateOrCreate(
                ['key' => $key],
                ['value' => is_array($value) ? $value : $value]
            );
        }
    }
}
```

### SiteController@store Update

File: `app/Http/Controllers/Admin/SiteController.php` (modify existing)

In the `store` method, after creating the site:

```php
// Add theme_id to validation
$validated = $request->validate([
    // ... existing rules
    'theme_id' => 'nullable|exists:themes,id',
]);

// After site creation, if theme is selected, clone blocks_preset
if ($request->filled('theme_id')) {
    $theme = Theme::find($request->theme_id);
    $site->update([
        'theme_id' => $theme->id,
        'blocks' => $theme->blocks_preset,
    ]);
}
```

Also pass themes to the create view:

```php
public function create()
{
    $themes = Theme::active()->get();
    return view('admin.sites.create', compact('themes'));
}
```
