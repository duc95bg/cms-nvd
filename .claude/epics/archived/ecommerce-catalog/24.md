## Size: M (3h)

## Description

Product detail page + variant selection UX + JSON API endpoint for realtime price/stock.

## Files

- Add `show()` to `app/Http/Controllers/ProductController.php` (public)
- `app/Http/Controllers/Api/VariantPriceController.php` — single `__invoke()` endpoint
- `resources/views/products/show.blade.php` — full detail page

## Key details

- **Gallery:** main image + thumbnails, click thumbnail to swap main (vanilla JS)
- **Attribute selectors:** one `<select>` or button group per attribute. When all attributes selected → fetch `/api/product/{id}/variant-price?values[]=1&values[]=3` → update price, stock badge, SKU display, variant image
- **API response:** `{ price: "250.000₫", raw_price: 250000, stock: 15, sku: "AT-M-DO", image: "/storage/...", in_stock: true }`
- **Add-to-cart button:** `<button data-product-id="{id}" data-variant-id="{variant_id}" disabled|enabled>`. Disabled when stock = 0 or no variant selected.
- **Breadcrumb:** Home > Category > Product
- **Related products:** same category, max 4, reuse `_card.blade.php` partial
- **SEO:** title from `$product->t('name')`, meta description from `$product->t('short_description')`

## Acceptance criteria

- [ ] `/{locale}/product/{slug}` renders full detail
- [ ] Selecting all attributes triggers fetch and updates price/stock/SKU
- [ ] API returns correct variant data
- [ ] Add-to-cart button correctly enabled/disabled
- [ ] Gallery image swap works
- [ ] Breadcrumb and related products render
- [ ] SEO meta tags present

## Definition of Done

Code implemented, views compile, variant selector works with JS
