# Task: Wire public and admin routes

## Description
Add the public site render route into the existing `{locale}` group and register the full `admin.sites.*` prefix group pointing at `SiteController`. Depends on the controller landing first because the route file imports the class.

## Acceptance Criteria
- [ ] `routes/web.php` imports `use App\Http\Controllers\SiteController;`
- [ ] Public route added **inside** the existing `Route::prefix('{locale}')->where(['locale' => 'en|vi'])` group:
  ```php
  Route::get('/site/{slug}', [SiteController::class, 'show'])
      ->where('slug', '[a-z0-9\-]+')
      ->name('site.show');
  ```
- [ ] New admin group outside the locale prefix:
  ```php
  Route::prefix('admin')
      ->name('admin.')
      ->middleware(['web' /*, 'auth' */])
      ->group(function () { ... });
  ```
  with these routes:
  - `GET sites` → `index` → `admin.sites.index`
  - `GET sites/create` → `create` → `admin.sites.create`
  - `POST sites` → `store` → `admin.sites.store`
  - `GET sites/{site}/edit` → `edit` → `admin.sites.edit`
  - `PUT sites/{site}` → `update` → `admin.sites.update`
  - `GET sites/{site}/preview` → `preview` → `admin.sites.preview`
  - `POST sites/{site}/images` → `uploadImage` → `admin.sites.images.upload`
- [ ] `auth` middleware left commented with a note explaining it lands when Breeze/Jetstream is added
- [ ] `php artisan route:list` shows all 8 routes (1 public + 7 admin) without errors

## Technical Details
- File: `routes/web.php`
- Do not touch the existing `/`, `home`, `about` routes from the `multi-language` epic.
- Do not add a new middleware group — reuse the existing `web` group. `SetLocale` is already appended from the multi-language epic and handles the `{locale}` segment.
- Route-model binding `{site}` uses the default `id` column — no custom binding needed.
