# Task: Admin create site Blade view

## Description
The first screen an authenticated user sees when spinning up a new site: pick a template from a `<select>` and type a slug. Form posts to `admin.sites.store`, which is handled in task 11.

## Acceptance Criteria
- [ ] `resources/views/admin/sites/create.blade.php` exists
- [ ] Minimal standalone Blade (own `<html>`, own `<head>`, Tailwind CDN) — not under the template layout, this is admin UI
- [ ] Form posts to `route('admin.sites.store')` with `@csrf`
- [ ] `<select name="template_id">` loops over `$templates` showing `{name} ({type})`
- [ ] `<input name="slug">` with `required`, `maxlength=80`, `pattern="[a-z0-9\-]+"`, helper text explaining the constraint
- [ ] Validation errors displayed under each field via `@error(...)`
- [ ] Submit button labeled with `{{ __('Create site') }}` so it localizes through the existing lang files
- [ ] All human-readable strings wrapped in `__()` for localization

## Technical Details
- File: `resources/views/admin/sites/create.blade.php`
- Expects `$templates` Eloquent collection from `SiteController@create`.
- Add any new lang keys used in this view to `lang/en.json` and `lang/vi.json`:
  - `Create site`, `Create a new site`, `Template`, `Slug`, `Lowercase letters, numbers and dashes only.`
- No JS needed.
