## Size: M (2.5h)

## Description

Resource CRUD for attributes + inline attribute values management.

## Files

- `app/Http/Controllers/Admin/AttributeController.php` — index, create, store, edit, update, destroy
- `resources/views/admin/attributes/index.blade.php` — table listing
- `resources/views/admin/attributes/create.blade.php` — form: name[en], name[vi], type select (select/color/text), dynamic "add value" rows
- `resources/views/admin/attributes/edit.blade.php` — same, pre-filled, can add/remove values

## Key detail

Attribute values are managed inline in the attribute form, not as a separate page. Vanilla JS to dynamically add/remove value rows (`appendChild`/`removeChild`). Each value row: value[en], value[vi], sort_order. On save, controller syncs values (create new, update existing, delete removed).

## Acceptance criteria

- [ ] Create attribute "Kích thước" with values S, M, L, XL
- [ ] Edit: add new value, remove existing, reorder
- [ ] Delete attribute blocked if used by any product
- [ ] Inline JS for dynamic value rows (no Alpine, no framework)
- [ ] All strings via `__()`

Code implemented, views compile, inline values JS works
