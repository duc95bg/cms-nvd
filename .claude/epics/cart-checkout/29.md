# 29 — Data layer: migrations, models, config/payment.php

**Size:** S (1.5h)

## Description

Create the foundational data layer for the cart-checkout feature. This includes three database migrations for orders, order items, and order status logs; three Eloquent models with relationships, scopes, and helper methods; a `config/payment.php` configuration file for all payment provider settings; and an update to `.env.example` with placeholder values.

This task has no dependencies and can be worked on immediately in parallel with other independent tasks.

## Acceptance Criteria

1. Three migrations exist and run successfully via `php artisan migrate`.
2. The `orders` table schema matches PRD FR-1 exactly.
3. The `order_items` table stores product snapshots (name, price, variant info) so historical orders are not affected by product changes.
4. The `order_status_logs` table records every status transition with timestamp and the user who made the change.
5. All three models have correct fillable, casts, and relationships.
6. `Order::canTransitionTo($status)` enforces valid state machine transitions.
7. `config/payment.php` reads all values from `.env` with sensible defaults.
8. `.env.example` includes all new payment-related environment variables with placeholder values.

## Technical Details

### Migrations

**Migration 1: `create_orders_table`**

```php
Schema::create('orders', function (Blueprint $table) {
    $table->id();
    $table->string('order_number')->unique();
    $table->foreignId('user_id')->nullable()->constrained()->nullOnDelete();
    $table->string('customer_name');
    $table->string('customer_email');
    $table->string('customer_phone');
    $table->text('customer_address');
    $table->text('customer_notes')->nullable();
    $table->string('payment_method');       // cod, bank_transfer, vnpay, paypal
    $table->string('payment_status')->default('pending'); // pending, paid, failed, refunded
    $table->string('status')->default('pending');          // pending, confirmed, shipping, delivered, cancelled
    $table->decimal('subtotal', 15, 2);
    $table->decimal('total', 15, 2);
    $table->json('payment_data')->nullable(); // store gateway responses
    $table->timestamps();
});
```

**Migration 2: `create_order_items_table`**

```php
Schema::create('order_items', function (Blueprint $table) {
    $table->id();
    $table->foreignId('order_id')->constrained()->cascadeOnDelete();
    $table->foreignId('product_id')->nullable()->constrained()->nullOnDelete();
    $table->foreignId('variant_id')->nullable();
    $table->string('product_name');           // snapshot
    $table->decimal('product_price', 15, 2);  // snapshot
    $table->json('variant_info')->nullable();  // snapshot of variant attributes
    $table->integer('quantity');
    $table->decimal('line_total', 15, 2);
    $table->timestamps();
});
```

**Migration 3: `create_order_status_logs_table`**

```php
Schema::create('order_status_logs', function (Blueprint $table) {
    $table->id();
    $table->foreignId('order_id')->constrained()->cascadeOnDelete();
    $table->string('old_status');
    $table->string('new_status');
    $table->text('note')->nullable();
    $table->foreignId('changed_by')->nullable()->constrained('users')->nullOnDelete();
    $table->timestamps();
});
```

### Models

**`app/Models/Order.php`**

- `$fillable`: order_number, user_id, customer_name, customer_email, customer_phone, customer_address, customer_notes, payment_method, payment_status, status, subtotal, total, payment_data
- `$casts`: payment_data as array, subtotal as decimal:2, total as decimal:2
- Relationships: `items()` hasMany OrderItem, `statusLogs()` hasMany OrderStatusLog, `user()` belongsTo User
- Scopes: `scopeForUser($query, $userId)`, `scopeByStatus($query, $status)`
- Method: `canTransitionTo(string $newStatus): bool` — enforces state machine:
  - pending → confirmed, cancelled
  - confirmed → shipping, cancelled
  - shipping → delivered
  - delivered → (no transitions)
  - cancelled → (no transitions)
- Method: `generateOrderNumber(): string` — format: `ORD-{YYYYMMDD}-{random 6 chars uppercase}`. Called in OrderService, not in model boot (to keep model simple).

**`app/Models/OrderItem.php`**

- `$fillable`: order_id, product_id, variant_id, product_name, product_price, variant_info, quantity, line_total
- `$casts`: variant_info as array, product_price as decimal:2, line_total as decimal:2
- Relationships: `order()` belongsTo Order, `product()` belongsTo Product (nullable)

**`app/Models/OrderStatusLog.php`**

- `$fillable`: order_id, old_status, new_status, note, changed_by
- Relationships: `order()` belongsTo Order, `changedBy()` belongsTo User (foreign key: changed_by)
