# 37 — Wire all routes

**Size:** XS (0.5h)

## Description

Register all cart, checkout, payment, order, and admin order routes in `routes/web.php`. Configure CSRF exclusions for payment callback routes. Verify all routes are registered correctly with `php artisan route:list`.

This task depends on all controller tasks (31-36) being complete so that all referenced controllers and methods exist.

## Acceptance Criteria

1. All cart routes are registered and functional.
2. All checkout routes are registered and functional.
3. All payment callback routes are registered with CSRF exclusion.
4. All user order routes are registered with auth middleware.
5. All admin order routes are registered with admin middleware.
6. `php artisan route:list` shows all new routes without errors.
7. No route name conflicts with existing routes.

## Technical Details

### Route definitions for `routes/web.php`

```php
use App\Http\Controllers\CartController;
use App\Http\Controllers\CheckoutController;
use App\Http\Controllers\PaymentController;
use App\Http\Controllers\OrderController;
use App\Http\Controllers\Admin\OrderController as AdminOrderController;

// ──────────────────────────────────────────────
// Cart routes (no auth required — guest cart)
// ──────────────────────────────────────────────
Route::post('/cart/add', [CartController::class, 'add'])->name('cart.add');
Route::get('/cart', [CartController::class, 'index'])->name('cart.index');
Route::post('/cart/update', [CartController::class, 'update'])->name('cart.update');
Route::post('/cart/remove', [CartController::class, 'remove'])->name('cart.remove');

// ──────────────────────────────────────────────
// Checkout routes (no auth required — guest checkout)
// ──────────────────────────────────────────────
Route::get('/checkout', [CheckoutController::class, 'index'])->name('checkout.index');
Route::post('/checkout', [CheckoutController::class, 'process'])->name('checkout.process');

// ──────────────────────────────────────────────
// Payment callback routes (no auth, no CSRF)
// ──────────────────────────────────────────────
Route::get('/payment/vnpay/return', [PaymentController::class, 'vnpayReturn'])
    ->name('payment.vnpay.return');
Route::get('/payment/vnpay/ipn', [PaymentController::class, 'vnpayIpn'])
    ->name('payment.vnpay.ipn')
    ->withoutMiddleware([\App\Http\Middleware\VerifyCsrfToken::class]);
Route::get('/payment/paypal/return', [PaymentController::class, 'paypalReturn'])
    ->name('payment.paypal.return');
Route::get('/payment/paypal/cancel', [PaymentController::class, 'paypalCancel'])
    ->name('payment.paypal.cancel');

// ──────────────────────────────────────────────
// Order result pages (no auth — accessed via redirect after checkout)
// ──────────────────────────────────────────────
Route::get('/orders/{order}/success', [OrderController::class, 'success'])
    ->name('orders.success');
Route::get('/orders/{order}/bank-transfer', [OrderController::class, 'bankTransfer'])
    ->name('orders.bank-transfer');

// ──────────────────────────────────────────────
// User order history (auth required)
// ──────────────────────────────────────────────
Route::middleware('auth')->group(function () {
    Route::get('/orders', [OrderController::class, 'history'])
        ->name('orders.history');
    Route::get('/orders/{order}/detail', [OrderController::class, 'detail'])
        ->name('orders.detail');
});

// ──────────────────────────────────────────────
// Admin order management (auth + admin middleware)
// ──────────────────────────────────────────────
// Add inside the existing admin route group:
Route::middleware(['auth', 'admin'])->prefix('admin')->name('admin.')->group(function () {
    Route::get('/orders', [AdminOrderController::class, 'index'])
        ->name('orders.index');
    Route::get('/orders/{order}', [AdminOrderController::class, 'show'])
        ->name('orders.show');
    Route::post('/orders/{order}/status', [AdminOrderController::class, 'updateStatus'])
        ->name('orders.update-status');
});
```

**Notes on integration:**
- If the project already has an admin route group, add the admin order routes inside it rather than creating a duplicate group.
- Adjust middleware names (`admin`, `is_admin`, `role:admin`, etc.) to match the project's existing admin middleware.
- Use the same route prefix/naming conventions as existing admin routes.

### CSRF exclusion

Two options depending on what was done in tasks 33/34:

**Option A: Already handled via `VerifyCsrfToken` middleware `$except`:**
```php
// In app/Http/Middleware/VerifyCsrfToken.php
protected $except = [
    'payment/vnpay/*',
    'payment/paypal/*',
];
```

**Option B: Route-level exclusion (as shown above):**
Use `->withoutMiddleware()` on individual routes. This is preferred for GET routes that don't normally need CSRF (VNPay IPN is a GET).

For the VNPay IPN specifically, CSRF exclusion is critical because VNPay's server makes the request (no session/cookie). The return URLs are GET requests from the user's browser and typically don't need CSRF either, but exclude them to be safe.
