# 35 — Admin order management

**Size:** M (2.5h)

## Description

Build the admin interface for managing orders: a paginated order list with status filtering and search, an order detail view with items and status history, and a status update action that enforces valid transitions and logs changes.

This task depends on task 29 (models must exist). It can be worked on in parallel with tasks 31, 33, 34, 36.

## Acceptance Criteria

1. Admin can view a paginated list of all orders.
2. Admin can filter orders by status using a dropdown.
3. Admin can search orders by order_number, customer_name, or customer_email.
4. Admin can view order details including customer info, items, payment info, and status history.
5. Admin can update order status using a dropdown that only shows valid transitions.
6. Invalid status transitions are rejected with an error message.
7. Every status change is logged in the order_status_logs table with the admin user who made the change.
8. Status badges use distinct colors for each status.

## Technical Details

### `app/Http/Controllers/Admin/OrderController.php`

```php
namespace App\Http\Controllers\Admin;

class OrderController extends Controller
{
    /**
     * GET /admin/orders
     * Paginated order list with optional status filter and search.
     */
    public function index(Request $request): View
    {
        $query = Order::with('user')
            ->latest();

        // Status filter
        if ($request->filled('status')) {
            $query->byStatus($request->status);
        }

        // Search: order_number, customer_name, customer_email
        if ($request->filled('search')) {
            $search = $request->search;
            $query->where(function ($q) use ($search) {
                $q->where('order_number', 'like', "%{$search}%")
                  ->orWhere('customer_name', 'like', "%{$search}%")
                  ->orWhere('customer_email', 'like', "%{$search}%");
            });
        }

        $orders = $query->paginate(20)->withQueryString();

        return view('admin.orders.index', [
            'orders'         => $orders,
            'currentStatus'  => $request->status,
            'search'         => $request->search,
            'statuses'       => ['pending', 'confirmed', 'shipping', 'delivered', 'cancelled'],
        ]);
    }

    /**
     * GET /admin/orders/{order}
     * Order detail with items and status logs.
     */
    public function show(Order $order): View
    {
        $order->load(['items', 'statusLogs.changedBy', 'user']);

        // Determine allowed transitions for the status update form
        $allowedTransitions = collect(['pending', 'confirmed', 'shipping', 'delivered', 'cancelled'])
            ->filter(fn ($status) => $order->canTransitionTo($status))
            ->values();

        return view('admin.orders.show', [
            'order'              => $order,
            'allowedTransitions' => $allowedTransitions,
        ]);
    }

    /**
     * POST /admin/orders/{order}/status
     * Update order status with validation and logging.
     */
    public function updateStatus(Request $request, Order $order): RedirectResponse
    {
        $validated = $request->validate([
            'status' => 'required|string|in:pending,confirmed,shipping,delivered,cancelled',
            'note'   => 'nullable|string|max:500',
        ]);

        if (!$order->canTransitionTo($validated['status'])) {
            return back()->with('error', __('Cannot transition to this status'));
        }

        $oldStatus = $order->status;

        $order->update(['status' => $validated['status']]);

        OrderStatusLog::create([
            'order_id'   => $order->id,
            'old_status' => $oldStatus,
            'new_status' => $validated['status'],
            'note'       => $validated['note'] ?? null,
            'changed_by' => auth()->id(),
        ]);

        return back()->with('success', __('Status updated'));
    }
}
```
