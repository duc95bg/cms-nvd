# 30 — Services: CartService, OrderService, PaymentService

**Size:** M (3h)

## Description

Create the three core service classes that power the cart-checkout feature: CartService for session-based cart management, OrderService for order creation with stock management, and PaymentService for VNPay and PayPal payment URL generation and verification. Also create the InsufficientStockException and register CartService as a singleton.

This task depends on task 29 (models and config must exist first).

## Acceptance Criteria

1. CartService manages cart state in the session with full CRUD operations.
2. Cart items are keyed by `{productId}-{variantId}` to support multiple variants of the same product.
3. Adding or updating cart items validates stock availability and throws InsufficientStockException when stock is insufficient.
4. OrderService creates orders in a database transaction with row-level locking to prevent race conditions on stock.
5. OrderService decrements variant/product stock atomically and snapshots product data into order items.
6. PaymentService generates valid VNPay payment URLs with correct HMAC-SHA512 hash.
7. PaymentService creates PayPal orders via REST API v2 and returns approval URLs.
8. CartService is registered as a singleton in AppServiceProvider.

## Technical Details

### `app/Services/CartService.php`

Session key: `"cart"`. Cart data structure in session is an associative array keyed by `"{productId}-{variantId}"`.

```php
class CartService
{
    /**
     * Add item to cart. If item already exists, increment quantity.
     * Validates stock availability. Throws InsufficientStockException if insufficient.
     */
    public function add(int $productId, ?int $variantId, int $qty = 1): void

    /**
     * Update quantity for a cart item by its key ("{productId}-{variantId}").
     * If qty <= 0, removes the item. Validates stock.
     */
    public function update(string $key, int $qty): void

    /**
     * Remove an item from cart by key.
     */
    public function remove(string $key): void

    /**
     * Clear entire cart.
     */
    public function clear(): void

    /**
     * Get all cart items as a Collection of stdClass objects.
     * Each object includes: key, product_id, variant_id, quantity,
     * plus loaded product (with images) and variant (if applicable),
     * plus computed unit_price and line_total.
     */
    public function items(): Collection

    /**
     * Calculate grand total of all items.
     */
    public function total(): float

    /**
     * Count total number of items (sum of quantities).
     */
    public function count(): int

    /**
     * Check if cart is empty.
     */
    public function isEmpty(): bool
}
```

Internal storage per item in session:
```php
[
    'product_id' => int,
    'variant_id' => int|null,
    'quantity'   => int,
]
```

Stock validation logic:
- If variant_id is set, check the variant's stock column.
- If variant_id is null, check the product's stock column.
- Compare requested quantity (existing qty + new qty for add) against available stock.

### `app/Exceptions/InsufficientStockException.php`

```php
class InsufficientStockException extends \RuntimeException
{
    public function __construct(
        public readonly string $productName,
        public readonly int $requestedQty,
        public readonly int $availableStock
    ) {
        parent::__construct(
            "Insufficient stock for {$productName}: requested {$requestedQty}, available {$availableStock}"
        );
    }
}
```

### `app/Services/OrderService.php`

```php
class OrderService
{
    /**
     * Create an order from the current cart contents.
     *
     * @param array $customer Keys: name, email, phone, address, notes
     * @param string $paymentMethod One of: cod, bank_transfer, vnpay, paypal
     * @param CartService $cart
     * @return Order The created order with items loaded
     *
     * @throws InsufficientStockException if stock changed between cart and checkout
     */
    public function createOrder(array $customer, string $paymentMethod, CartService $cart): Order
}
```

Implementation details:
1. Wrap in `DB::transaction()`.
2. For each cart item, `lockForUpdate()` the variant (or product) row to prevent race conditions.
3. Re-validate stock under lock. Throw InsufficientStockException if insufficient.
4. Decrement stock on variant/product.
5. Create Order with:
   - `order_number`: generated via `Order::generateOrderNumber()` or inline: `'ORD-' . date('Ymd') . '-' . strtoupper(Str::random(6))`
   - `user_id`: `auth()->id()` (nullable for guest checkout)
   - Customer fields from `$customer` array
   - `payment_method`: from parameter
   - `payment_status`: 'pending'
   - `status`: 'pending'
   - `subtotal` and `total`: calculated from cart items
6. Create OrderItem for each cart item with snapshots:
   - `product_name`: from product
   - `product_price`: from variant price or product price
   - `variant_info`: JSON of variant attributes (e.g., `{"color": "Red", "size": "XL"}`)
   - `quantity`, `line_total`
7. Create initial OrderStatusLog: old_status='', new_status='pending', note='Order created'.
8. Clear the cart via `$cart->clear()`.
9. Return the order with items loaded.

### `app/Services/PaymentService.php`

```php
class PaymentService
{
    /**
     * Generate VNPay payment URL for the given order.
     */
    public static function createVnpayUrl(Order $order): string

    /**
     * Verify VNPay return/IPN hash.
     * @return bool True if hash is valid
     */
    public static function verifyVnpayHash(array $params): bool

    /**
     * Create a PayPal order and return the approval URL.
     */
    public static function createPaypalOrder(Order $order): string

    /**
     * Capture a PayPal payment after user approval.
     * @return array PayPal capture response data
     */
    public static function capturePaypalPayment(string $token): array

    /**
     * Get PayPal OAuth2 access token.
     */
    private static function getPaypalAccessToken(): string
}
```

**VNPay URL generation:**
1. Build params array: vnp_Version=2.1.0, vnp_Command=pay, vnp_TmnCode, vnp_Amount (total * 100, integer), vnp_CurrCode=VND, vnp_TxnRef=order_number, vnp_OrderInfo, vnp_OrderType=other, vnp_Locale=vn, vnp_ReturnUrl, vnp_IpAddr, vnp_CreateDate (YmdHis).
2. Sort params by key (`ksort`).
3. Build query string from sorted params.
4. Generate HMAC-SHA512 hash using hash_secret.
5. Append `&vnp_SecureHash={hash}` to URL.

**VNPay hash verification:**
1. Extract `vnp_SecureHash` from params, remove it and `vnp_SecureHashType` from array.
2. Sort remaining params by key.
3. Build query string, compute HMAC-SHA512, compare with extracted hash.

**PayPal order creation:**
1. Get access token via `getPaypalAccessToken()`.
2. Convert VND total to USD: `round($order->total / config('payment.paypal.usd_rate'), 2)`.
3. POST to `/v2/checkout/orders` with:
   - intent: CAPTURE
   - purchase_units with amount in USD
   - application_context with return_url and cancel_url
4. Return the "approve" link from response.

**PayPal base URL:** `config('payment.paypal.mode') === 'sandbox' ? 'https://api-m.sandbox.paypal.com' : 'https://api-m.paypal.com'`

**PayPal access token:** POST to `/v1/oauth2/token` with Basic Auth (client_id:secret), body `grant_type=client_credentials`.
