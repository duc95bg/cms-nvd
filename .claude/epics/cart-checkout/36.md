# 36 — User order history

**Size:** S (1.5h)

## Description

Build the user-facing order history: a list of the authenticated user's orders and a detail page for a single order. Both pages require authentication and the detail page enforces ownership (users can only view their own orders).

This task depends on task 29 (models must exist). It can be worked on in parallel with tasks 31, 33, 34, 35.

## Acceptance Criteria

1. Authenticated users can view their order history at `/orders`.
2. The order list shows order number, date, total, and status with colored badges.
3. Users can click through to view details of a specific order.
4. Users can only view their own orders (ownership check).
5. Unauthenticated users are redirected to login.
6. Empty state is handled when user has no orders.
7. Order detail shows items, shipping info, payment status, and status timeline.

## Technical Details

### OrderController additions

Add these methods to `app/Http/Controllers/OrderController.php` (created in task 32):

```php
/**
 * GET /orders
 * List authenticated user's orders, newest first.
 * Requires authentication (middleware 'auth').
 */
public function history(): View
{
    $orders = Order::forUser(auth()->id())
        ->latest()
        ->paginate(15);

    return view('orders.history', compact('orders'));
}

/**
 * GET /orders/{order}
 * Show single order detail.
 * Requires authentication + ownership check.
 */
public function detail(Order $order): View
{
    // Ownership check
    if ($order->user_id !== auth()->id()) {
        abort(403);
    }

    $order->load(['items', 'statusLogs']);

    return view('orders.detail', compact('order'));
}
```

Both methods should be protected by the `auth` middleware (applied at the route level or in the controller constructor).
