# 31 — Cart UI: CartController, views, badge, wire product detail button

**Size:** M (2h)

## Description

Build the cart user interface: the CartController with add/index/update/remove actions, the cart page view, a cart badge partial for the header, and wiring the "Add to Cart" button on the product detail page to actually submit a form.

This task depends on task 30 (CartService must exist). It can be worked on in parallel with tasks 33, 34, 35, 36 once 30 is complete.

## Acceptance Criteria

1. Users can add products to cart from the product detail page via a POST form.
2. The cart page displays all items with image, name, variant attributes, unit price, quantity input, line total, and remove button.
3. Users can update quantity or remove items from the cart page.
4. A cart badge in the header shows the current item count.
5. Empty cart state is handled gracefully with a message and "Continue shopping" link.
6. Flash messages appear on add/update/remove actions.
7. Stock validation errors are caught and displayed to the user.

## Technical Details

### `app/Http/Controllers/CartController.php`

```php
class CartController extends Controller
{
    public function __construct(private CartService $cart) {}

    /**
     * POST /cart/add
     * Validates: product_id (required|exists:products,id), variant_id (nullable|integer), qty (required|integer|min:1)
     * Calls CartService::add(), catches InsufficientStockException.
     * Redirects back with success/error flash message.
     */
    public function add(Request $request): RedirectResponse

    /**
     * GET /cart
     * Shows the cart page with items and total.
     */
    public function index(): View

    /**
     * POST /cart/update
     * Validates: key (required|string), qty (required|integer|min:0)
     * Calls CartService::update(). qty=0 removes the item.
     * Redirects back with flash.
     */
    public function update(Request $request): RedirectResponse

    /**
     * POST /cart/remove
     * Validates: key (required|string)
     * Calls CartService::remove().
     * Redirects back with flash.
     */
    public function remove(Request $request): RedirectResponse
}
```

Error handling for `add()` and `update()`:
```php
try {
    $this->cart->add($request->product_id, $request->variant_id, $request->qty);
    return back()->with('success', __('Added to cart'));
} catch (InsufficientStockException $e) {
    return back()->with('error', __('Insufficient stock') . ': ' . $e->getMessage());
}
```

### `resources/views/cart/index.blade.php`

- Extends `layouts.catalog` (or whatever the storefront layout is).
- Page title: `__('Shopping cart')`
- If cart is empty:
  - Show empty state message: `__('Your cart is empty')`
  - "Start shopping" link to product catalog
- If cart has items, show a table/list:
  - **Column: Image** — product thumbnail (first image)
  - **Column: Product** — product name + variant attributes (e.g., "Color: Red, Size: XL")
  - **Column: Unit Price** — formatted price
  - **Column: Quantity** — `<input type="number">` inside a `<form>` that POSTs to `/cart/update` with hidden `key` field. Include a small "Update" button.
  - **Column: Line Total** — unit_price * quantity, formatted
  - **Column: Remove** — `<form>` POSTing to `/cart/remove` with hidden `key` field. Red "X" or trash icon button.
- Below the table:
  - **Grand Total** displayed prominently: `{{ number_format($total, 0, ',', '.') }}đ`
- Action buttons:
  - "Continue shopping" link (secondary) to catalog
  - "Checkout" link/button (primary) to `/checkout`
