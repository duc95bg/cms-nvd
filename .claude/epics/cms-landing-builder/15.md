# Task: Feature and unit tests for the CMS flow

## Description
End-to-end coverage for the epic. One feature test class covers the public render path and the admin happy path; one unit test pins down the flatten helper behaviour that drives the edit form. All tests use `RefreshDatabase` + `TemplateSeeder` so they're hermetic.

## Acceptance Criteria

### `tests/Feature/SiteCmsTest.php`
- [ ] Uses `RefreshDatabase` trait
- [ ] `setUp()` seeds the `product` template via `$this->seed(TemplateSeeder::class)` and creates a test user via factory
- [ ] `test_public_show_renders_vietnamese_content` — creates a published site, `GET /vi/site/<slug>`, asserts 200 + sees the Vietnamese hero title
- [ ] `test_public_show_renders_english_content` — same site, `GET /en/site/<slug>`, asserts English hero title
- [ ] `test_public_show_returns_404_for_unpublished_site` — unpublished site, `GET /vi/site/<slug>`, asserts 404
- [ ] `test_public_show_returns_404_for_unsupported_locale` — `GET /fr/site/<slug>`, asserts 404 (route constraint)
- [ ] `test_admin_create_clones_default_content` — authenticated as user, `POST admin/sites` with `{template_id, slug}`, asserts redirect to edit, asserts the new `sites.content` deep-equals the template's `default_content`
- [ ] `test_admin_update_rehydrates_dot_notation_into_nested_json` — `PUT admin/sites/{id}` with `content[hero.title][vi]=X` and `content[hero.title][en]=Y`, asserts `$site->fresh()->content['hero']['title']['vi'] === 'X'` and `['en'] === 'Y'`
- [ ] `test_admin_preview_renders_regardless_of_published` — unpublished site, `GET admin/sites/{id}/preview` as owner, asserts 200
- [ ] `test_admin_routes_reject_non_owner` — second user tries `GET admin/sites/{id}/edit`, asserts 403
- [ ] `test_image_upload_stores_file_and_returns_url` — `POST admin/sites/{id}/images` with `UploadedFile::fake()->image('a.jpg')`, asserts 200 + JSON has `url`, asserts `Media` row created, asserts file exists on the `public` disk

### `tests/Unit/SiteContentFlattenTest.php`
- [ ] Replicates the `$flatten` closure from the edit view as a standalone pure function (or extracts it into a helper and imports here — your call)
- [ ] Test: scalar value at root returns `['key' => value]`
- [ ] Test: translatable leaf `['en' => 'x', 'vi' => 'y']` at `hero.title` returns `['hero.title' => ['en' => 'x', 'vi' => 'y']]`
- [ ] Test: nested group `['hero' => ['title' => ['en' => 'x', 'vi' => 'y'], 'cta_url' => '#']]` flattens to `['hero.title' => [...], 'hero.cta_url' => '#']`
- [ ] Test: indexed list `['features' => ['items' => [{...}, {...}]]]` returns `['features.items' => [...]]` (list preserved as leaf)

## Technical Details
- Files:
  - `tests/Feature/SiteCmsTest.php`
  - `tests/Unit/SiteContentFlattenTest.php`
- Use `Storage::fake('public')` in the image upload test.
- For the "second user" ownership test, create the user with `User::factory()` — need a factory; the default Laravel install ships one.
- The controller does `request()->user()?->id` for ownership — feature tests must `actingAs($user)` before hitting admin routes.
- If the `$flatten` closure lives inside the Blade file, the unit test can either duplicate it or extract to `app/Support/FlattenContent.php`. Prefer extraction so task 13's view imports the helper too — this keeps one source of truth.
