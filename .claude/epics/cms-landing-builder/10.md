# Task: Base layout and product Blade template

## Description
Build the shared Tailwind layout that all templates extend, and the first production template (`product`) that renders hero + features repeater + pricing CTA using the `Site::t()` helper. No controller work here — this task is pure Blade.

## Acceptance Criteria
- [ ] `resources/views/templates/layouts/base.blade.php` exists with:
  - `<html lang="{{ app()->getLocale() }}">`
  - Tailwind via CDN (`<script src="https://cdn.tailwindcss.com"></script>`)
  - Header including `@include('partials.lang-switcher')` (reuses the partial from the multi-language epic)
  - `<title>` uses `$site->t('seo.title', null, $site->slug)`
  - `<meta name="description">` uses `$site->t('seo.description')`
  - Footer with brand name and current year
  - `@yield('content')` in `<main>`
- [ ] `resources/views/templates/product.blade.php` extends base and renders:
  - Hero section: `$site->t('hero.title')`, `$site->t('hero.subtitle')`, CTA button with `$site->t('hero.cta_label')` linking to `$site->t('hero.cta_url', null, '#features')`
  - Features section: heading `$site->t('features.heading')` + `@foreach (data_get($site->content, 'features.items', []) as $feature)` each rendering `data_get($feature, 'title.'.app()->getLocale())` and `data_get($feature, 'body.'.app()->getLocale())`
  - Pricing section: heading, subheading, CTA — all via `$site->t()`
- [ ] No direct `app()->getLocale()` calls for translatable text — use `$site->t()` (the `data_get` in the features loop is intentional because `features.items` is an indexed array)

## Technical Details
- Files to create:
  - `resources/views/templates/layouts/base.blade.php`
  - `resources/views/templates/product.blade.php`
- Tailwind classes only — no custom CSS files.
- The language switcher partial `partials.lang-switcher` already exists from the multi-language epic; just include it.
- Viewing the template in a browser requires a Site model + public route, both of which land in later tasks — for this task, visual verification can be deferred to task 15 (feature tests) or done manually once task 11 lands.
