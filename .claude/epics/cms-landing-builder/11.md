# Task: SiteController with all actions (public + admin)

## Description
A single controller with all public and admin actions for sites. Keeping every action in one file avoids the parallel-edit conflict the epic would otherwise have between "create flow", "edit flow", "preview", and "upload" — they all touch the same class. Ownership is enforced inline via a private helper until auth + Policies land in a future epic.

## Acceptance Criteria
- [ ] `app/Http/Controllers/SiteController.php` exists with these public methods:
  - `show(string $locale, string $slug): View` — finds site by slug with `published=true` (404 otherwise), eager-loads `template`, returns `view($site->template->view, ['site' => $site])`
  - `index(Request $request): View` — paginated list of the current user's sites, 20 per page
  - `create(): View` — returns `admin.sites.create` with all templates ordered by type
  - `store(Request $request): RedirectResponse` — validates `template_id (exists)`, `slug (alpha_dash, max:80, unique)`; creates a site owned by `request()->user()`, clones `template.default_content` into `content`, `published=false`; redirects to `admin.sites.edit`
  - `edit(Site $site): View` — ownership check, returns `admin.sites.edit` with `$site` and `$locales = config('app.supported_locales', ['en','vi'])`
  - `update(Request $request, Site $site): RedirectResponse` — ownership check, validates `content (required, array)` and `published (sometimes, boolean)`, rehydrates nested array from dot-notation keys via `data_set`, saves, redirects back with flash `status`
  - `preview(Site $site): View` — ownership check, returns the template view regardless of `published`
  - `uploadImage(Request $request, Site $site): JsonResponse` — ownership check, validates `image (image, max:4096)`, stores under `sites/{site_id}` on `public` disk, creates `Media` row, returns `{id, url}`
- [ ] Private `authorizeSite(Site $site): void` helper: `abort_unless($site->user_id === request()->user()?->id, 403);`
- [ ] `update` dot-notation rehydration:
  ```php
  $nested = [];
  foreach ($request->input('content', []) as $dotKey => $value) {
      data_set($nested, $dotKey, $value);
  }
  ```

## Technical Details
- File: `app/Http/Controllers/SiteController.php`
- Do not introduce Form Request classes — inline `$request->validate()` is enough for this iteration.
- Do not use a Policy yet; Policies come in when auth lands.
- `show()` must gate on `published=true` — unpublished sites 404 publicly.
- The `users` table already exists from the default Laravel install; route-model binding on `{site}` works out of the box.
