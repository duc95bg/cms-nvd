# Task: Admin edit site Blade view with JSON-driven form

## Description
The edit screen auto-generates a form by walking the site's `content` tree. Translatable leaves (objects whose keys are supported locales) become side-by-side inputs, one per locale. Indexed lists (repeaters like `features.items`) become a raw JSON textarea — a pragmatic escape hatch rather than a half-built block editor. This is the hardest view in the epic because the flatten logic has to distinguish three leaf kinds cleanly.

## Acceptance Criteria
- [ ] `resources/views/admin/sites/edit.blade.php` exists
- [ ] Top of file: an inline PHP closure `$flatten` that takes `($data, $prefix = '')` and returns `[dotKey => value]` map
- [ ] Flatten rules:
  - Scalar → `[$prefix => $value]`
  - Indexed list (numeric keys 0..N) → treat as leaf, return `[$prefix => $value]`
  - Associative array whose keys intersect `$locales` → translatable leaf, return `[$prefix => $value]`
  - Otherwise → recurse into children, prepending keys with `$prefix.'.'`
- [ ] Form posts to `route('admin.sites.update', $site)` with `@csrf` and `@method('PUT')`
- [ ] For each `$fields` entry:
  - If value is translatable-leaf (assoc array, not list): grid of inputs, one per locale, `name="content[{{ $key }}][{{ $lang }}]"`
  - If value is an indexed list: `<textarea name="content[{{ $key }}]">` containing `json_encode($value, JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT)`
  - If value is scalar: single `<input name="content[{{ $key }}]">`
- [ ] `published` checkbox with `name="published" value="1"` reflecting `$site->published`
- [ ] Save button + link to `admin.sites.preview` opening in a new tab
- [ ] If `$site->published`, show a "View live" link to `/{locale}/site/{slug}` in a new tab
- [ ] Flash `session('status')` displayed at the top
- [ ] All strings wrapped in `__()` — add new lang keys to `en.json` and `vi.json`:
  - `Edit site`, `Edit`, `Preview`, `View live`, `Saved`, `Published`, `Save`, `Raw JSON list — edit carefully.`

## Technical Details
- File: `resources/views/admin/sites/edit.blade.php`
- Expects `$site` and `$locales` from controller.
- Note on JSON textarea: when the user edits and submits, the controller receives the raw JSON **string** at `content[key]`. The controller currently only does `data_set` on each top-level key, so the string lands as-is. Decoding it back into an array is out of scope for this task but should be noted in task 15's test plan — for now, list fields are effectively read-only unless the user knows JSON.
- The reason for a JSON textarea over a dedicated repeater UI: avoids a JS framework in this epic. A follow-up epic can upgrade list fields with Alpine.js.
