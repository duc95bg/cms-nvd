## Description

Create the `BlockRenderer` service that takes an array of block definitions and renders them into concatenated HTML using the Blade partials from task 42. Update `SiteController@show` to use block-based rendering when a site has blocks defined, falling back to the existing template-based rendering otherwise. ## Technical Details

### BlockRenderer Service

File: `app/Services/BlockRenderer.php`

```php
namespace App\Services;

class BlockRenderer
{
    /**
     * Render an array of blocks into concatenated HTML.
     *
     * @param array $blocks Array of block definitions, each with 'type', 'content', 'order'
     * @return string Concatenated HTML
     */
    public function render(array $blocks): string
    {
        // Sort blocks by 'order' field
        usort($blocks, fn($a, $b) => ($a['order'] ?? 0) <=> ($b['order'] ?? 0));

        $html = '';

        foreach ($blocks as $block) {
            $type = $block['type'] ?? null;

            if (!$type) {
                continue;
            }

            $viewName = "blocks.{$type}";

            if (!view()->exists($viewName)) {
                continue; // skip unknown block types
            }

            $html .= view($viewName, ['block' => $block['content'] ?? []])->render();
        }

        return $html;
    }
}
```

### SiteController Update

File: `app/Http/Controllers/SiteController.php` (modify existing)

In the `show` method, add block-based rendering logic:

```php
public function show(Site $site)
{
    // If site has blocks defined, use block-based rendering
    if (!empty($site->blocks)) {
        $renderer = new \App\Services\BlockRenderer();
        $renderedBlocks = $renderer->render($site->blocks);

        return view('site.blocks-page', [
            'site' => $site,
            'renderedBlocks' => $renderedBlocks,
        ]);
    }

    // Fall back to template-based rendering
    return view($site->template->view, ['site' => $site]);
}
```

### Blocks Page Layout

File: `resources/views/site/blocks-page.blade.php`

```blade
@extends('layouts.catalog')

@section('title', $site->name)

@section('content')
    <main class="blocks-page">
        {!! $renderedBlocks !!}
    </main>
@endsection
```

The layout should:
- Extend the existing catalog layout (or equivalent base layout)
- Use SettingService for any global site branding (logo, site name) if needed in the layout header/footer
- Output the pre-rendered blocks HTML using unescaped `{!! !!}` since BlockRenderer already rendered Blade partials
- Include the site's name as the page title

### Integration Notes

- The `BlockRenderer` is a plain class (no interface needed at this stage) — can be refactored to a contract later if needed
- The controller should import `BlockRenderer` or resolve it from the container
- The `$site->blocks` check uses `!empty()` to handle both `null` and `[]` cases
- Blocks are sorted by their `order` field before rendering to ensure correct display order
